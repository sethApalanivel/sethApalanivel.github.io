Phishing Campaign Analytics — Role‑Based Dashboards (Power BI)

Problem

Security training needed targeted interventions; leadership needed real‑time visibility across CTRs, reporting, and completion.

Data

De‑identified campaign logs and training completion records (sanitised).

Approach

Model a simple funnel, create exec/ops/training views, add refresh cadence and action playbooks.

Results

Core metrics
Improved visibility into CTR spikes and late reports; operational follow‑ups prioritised.
Business impact
Higher training coverage and faster response to risky cohorts; supports continuous improvement.

Artifacts

Code repository · Notebook / report

What I’d improve next

  • Run A/B or geo‑holdout where feasible; track lift and guardrails.
  • Add monitoring for drift, fairness, and alerting on KPI regressions.
